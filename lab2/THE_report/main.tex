\section{Introduction}
This report goes through three tasks.
First task was about implementing stack with queue and vice versa.
Second task was about implementing linked list.
Third task was about solving Josephus problem and comparing quickness with vector and linked list with/without iterator. The main goal is to understand how implementation style affects speed of the algorithms.


\section{Methodology}
The report based both on analytical and practical methods.

\subsection{Analytical methods}
Before assignment, lectures' review was made on sorting algorithms, their various implementations and time complexities. 

\subsection{Practical methods}
Implemented algorithms were completely written by Ada. It was chosen because of the wish to learn a new language which was quite unfortunate.

\section{Results}

\subsubsection{Stack and Queue}
Folowing task had such specifications on creating:
\begin{itemize}
\item Stack made of 2 Queues
\item Stack made of 1 Queue
\item Queue made of 2 Stacks
\item Queue made of 1 Stack.
\end{itemize}

To start with the assignment, vanilla data structures of stack and queue were created. 
After implementing it, this was found:

1. If implementing one data structure with two others, it will take more space. However, if using queue that made of 2 stack theoretically queue can fully fill both stack efficiently. Meanwhile, using stack made of 2 queues can only fill one queue maximum, therefore space is used unefficiently.

2. If implementing one data structure into another through recursion, it does take less space and more managable. However, getting the data out takes more time for stack and queue.

The most efficient structure was queue implemented with two stacks. It uses space more efficiently, and after first retrieval of data, enequeue and dequeue took O(1).

1. Ada's Vector (array without bounds) and manual counting of the index;
